## 2024-05-24 - Missing Input Validation in Venturi3D XML Parsing
**Vulnerability:** `venturi3d.cpp` read the `Resolution` parameter directly from an XML file and used it as a divisor (`1./N`) without validation. A value of 0 or less would cause a division-by-zero crash or undefined behavior (negative mesh spacing).
**Learning:** External configuration files (XML, JSON) are untrusted user input just like CLI arguments. The assumption that a config file is "internal" or "safe" is dangerous, especially in simulations where parameters dictate memory allocation and physics stability.
**Prevention:** Validate all numerical parameters read from external configuration files immediately upon retrieval. Use `std::exit` or throw exceptions to fail fast before invalid values propagate to critical logic (like mesh generation).
